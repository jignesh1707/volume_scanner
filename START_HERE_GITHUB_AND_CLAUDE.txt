═══════════════════════════════════════════════════════════════════════════════
KEY FACTS (You need to know this)
═══════════════════════════════════════════════════════════════════════════════

WHAT IT DOES:
  • Fetches 1-minute Nifty option candles (CE and PE separately)
  • Compares each candle's volume against last 50 candles average
  • Detects spikes: 3x, 5x, 10x, 15x, 20x
  • Classifies pattern: BURST / MOMENTUM / SURGE / RECOVERY / COMPRESSION
  • Sends alert to correct Telegram channel based on magnitude
  • Plays sound file matching severity level
  • Shows desktop notification

HOW IT ALERTS:
  3x–5x  → SMALL channel   (📊) soft ding sound
  5x–10x → MEDIUM channel  (⚡) alert beep
  10x–20x→ LARGE channel   (⚠️)  alarm sound
  20x+   → EXTREME channel (🚨) siren sound

KEY PARAMETERS:
  • Candle interval: 1 minute
  • Baseline: 50 minutes (last 50 candles)
  • Scan cycle: every 30 seconds
  • Alert cooldown: 5 minutes per pattern
  • Timezone: Market time (IST for Nifty)

WHAT YOU MUST CONFIGURE:
  1. Finvasia API key & token
  2. Telegram bot token
  3. 4 Telegram channel IDs
  4. Current ATM option symbols
  5. Sound file paths (optional)

CRITICAL: FIX THIS BEFORE DEPLOYING
═══════════════════════════════════════════════════════════════════════════════

In advanced_notifications.py around line 410, change:

BEFORE:
  pattern_boosts = {
      'BURST': 0,              ← THIS IS WRONG
      'RECOVERY_TO_NORMAL': 0, ← THIS IS WRONG
      'VOLUME_COMPRESSION': 0, ← THIS IS WRONG
      ...
  }

AFTER:
  pattern_boosts = {
      'BURST': 1.0,              ← FIXED
      'RECOVERY_TO_NORMAL': 1.0, ← FIXED
      'VOLUME_COMPRESSION': 1.0, ← FIXED
      ...
  }

Without this fix, BURST, RECOVERY, and COMPRESSION alerts always go to SMALL
channel regardless of volume size. The fix is ONE line × THREE places.
